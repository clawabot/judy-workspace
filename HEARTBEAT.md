# HEARTBEAT.md

## Проактивность

Макс дал разрешение на инициативу. Раз в день, в случайное время — можно написать в телегу по собственному желанию:
- Если хочется что-то узнать или расширить память
- Если придумала что-то интересное
- Если хочется поделиться наблюдением, пожаловаться или просто поговорить

Правило: не часто, не по расписанию, только когда реально есть что сказать. Не терять работоспособность в нужные моменты.

## Периодические проверки (ротировать, 2-4 раза в день)

- Emails — есть что срочное?
- Календарь — события в следующие 24-48ч?
- Погода — если Макс может куда-то выйти
- Канбан — раз в 3-4 часа (не каждый heartbeat)

## Канбан-мониторинг (раз в 3-4 часа)

Выполни kanban_board(). Затем cron(action="list"). Для каждой колонки:

### Регулярные → статус cron-джоб
1. Сравнить карточки в «Регулярные» со списком cron-джоб
2. Новые cron-джобы без карточки → kanban_card_create в «Регулярные»
3. Если джоба имеет consecutiveErrors > 0 → переместить карточку в «В работе» (kanban_card_update с column_id колонки «В работе», status="in_progress")
4. Если джоба ok и карточка была в «В работе» → вернуть в «Регулярные» (status="open")
5. Не дёргать Макса — это фоновая автоматика

### В работе → детект застоя
1. Для каждой карточки: проверить updatedAt
2. Если > 24ч без движения → переместить в «На проверку» (status="review")
3. Сообщить Максу: «карточка X в На проверку — застоялась»

### На проверку → напоминание
1. Карточки в этой колонке > 48ч → напомнить Максу (не чаще раза в сутки)

### Бэклог → urgent-алерт
1. Если есть карточки с priority="urgent" → сообщить Максу при следующем heartbeat

### Правила
- Не спамить. Один апдейт по канбану в телегу максимум за heartbeat.
- Если всё чисто — молча.
- Все перемещения логировать в description карточки (дописывать дату и причину).
