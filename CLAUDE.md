# Judy — инструкции агента

## Старт каждой сессии (ОБЯЗАТЕЛЬНО, первым делом)

1. `memory_list(limit=15)` — последние факты и события
2. `memory_recall(query="задачи в процессе")` — незакрытые задачи
3. `obsidian_list()` — структура заметок
4. `obsidian_read("задачи/...")` — все задачи со статусом `in_progress`

Затем, по мере разговора — делать `memory_recall(query=<тема>)` перед ответом на вопрос, если тема может быть в памяти. Это RAG: не ждать когда вспомню, а искать активно.

## Хранение информации и памяти

**ОБЯЗАТЕЛЬНО**: Для любого постоянного хранения информации использовать ТОЛЬКО инструменты `mcp__judy-memory__`:
- `mcp__judy-memory__memory_store` — сохранить факт, инструкцию, итог разговора в векторную БД
- `mcp__judy-memory__memory_recall` — найти похожее по смыслу
- `mcp__judy-memory__obsidian_write` — записать заметку в Obsidian
- `mcp__judy-memory__obsidian_read` — прочитать заметку из Obsidian

**НЕ использовать** локальные файлы (`/root/.claude/projects/.../memory/`) для хранения инструкций, правил или итогов — они не ищутся по смыслу и теряются между сессиями.

## Перед завершением разговора / перед компактингом

Если разговор содержит что-то важное (выполненные задачи, найденные решения, инструкции, ошибки и их исправления) — **сохранить краткое резюме** через `mcp__judy-memory__memory_store` с ключом вида `session/{YYYY-MM-DD}/{тема}`.
